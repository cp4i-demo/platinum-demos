## Scenario overview

This demo shows how the IBM Cloud Pak for Business Automation enables straight-through processing. To illustrate this, a customer’s refund process is automated.  

The capabilities used in this demo are IBM workflow, IBM decisions, and operational intelligence.  

***

## Demo structure

Introduction
1. Model workflow
2. Execute workflow without straight-through processing
3. Change the workflow for straight-through processing
4. Execute workflow with straight-through processing
5. Monitor operational intelligence
6. Model decisions
7. Reduce manual work

Summary

## Before the demo

### Requisition instructions

Select a delivery option: 
1.	SaaS using the Cloud Pak for Business Automation as a Service demo environment (likely an IBMer). If you need to request access, follow the instructions here. Your environment is pre-deployed.  
2.	Install yourself using one of two options: <br/>
a.	Option 1: Bring your own environment
Access the [dba-refund-request](https://github.com/ibm-cloud-architecture/dba-refund-request) GitHub page to read technical architecture information and find deployment instructions.  
b.	Option 2: Reserve on TechZone
For IBMers and business partners that have access to TechZone, you can also reserve a Cloud Pak for Automation OpenShift demo environment via the [Straight-through processing TechZone page](https://techzone.ibm.com/collection/5ef5196556dfa7001728cb71#tab-1). 

Once your environment is ready, continue to the next steps below.
